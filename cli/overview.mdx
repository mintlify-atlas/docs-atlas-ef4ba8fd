## Installation

Build the CLI from source:

```bash
cargo build --release --bin provekit-cli
```

The compiled binary will be available at `target/release/provekit-cli`.

## Basic Workflow

The typical workflow for using ProveKit CLI:

1. **Prepare** a compiled Noir program:
   ```bash
   provekit-cli prepare circuit.json
   ```

2. **Prove** with input values:
   ```bash
   provekit-cli prove noir_proof_scheme.pkp inputs.toml
   ```

3. **Verify** the generated proof:
   ```bash
   provekit-cli verify noir_proof_scheme.pkv proof.np
   ```

## Global Options

The following options are available when built with the `tracy` feature:

<ParamField path="--tracy" type="boolean">
  Enable Tracy profiling
</ParamField>

<ParamField path="--tracy-allocations" type="number">
  Enable Tracy allocation tracking with specified stack depth (or 0 for no stack traces)
</ParamField>

<ParamField path="--tracy-keepalive" type="boolean">
  Keep the process alive after completion to allow Tracy to collect data
</ParamField>

## Getting Help

For help with any command:

```bash
provekit-cli --help
provekit-cli <command> --help
```

## Source Reference

The CLI implementation is located in:
- Main entry point: `tooling/cli/src/main.rs`
- Command definitions: `tooling/cli/src/cmd/mod.rs`
- Individual commands: `tooling/cli/src/cmd/<command>.rs`
