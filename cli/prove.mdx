The `prove` command generates a zero-knowledge proof for a prepared Noir program using provided input values.

## Command Syntax

```bash
provekit-cli prove <prover_path> <input_path> [OPTIONS]
```

## Arguments

<ParamField path="prover_path" type="string" required>
  Path to the prepared proof scheme (`.pkp` file generated by `prepare`)
</ParamField>

<ParamField path="input_path" type="string" required>
  Path to the input values file (`.toml` format)
</ParamField>

## Options

<ParamField path="--out, -o" type="string" default="./proof.np">
  Path to store the generated proof file
</ParamField>

## Input File Format

Input values should be provided in TOML format, matching your Noir program's parameters:

```toml
# Simple values
x = "5"
y = "10"

# Arrays
values = ["1", "2", "3", "4"]

# Structs
[config]
threshold = "100"
enabled = true
```

<Note>
Numeric values should be provided as strings to handle large field elements correctly.
</Note>

## Examples

### Basic Usage

Generate a proof with default output:

```bash
provekit-cli prove noir_proof_scheme.pkp inputs.toml
```

This creates `proof.np` in the current directory.

### Custom Output Path

Specify where to save the proof:

```bash
provekit-cli prove noir_proof_scheme.pkp inputs.toml --out ./proofs/my_proof.np
```

### Using Short Flags

```bash
provekit-cli prove scheme.pkp inputs.toml -o proof.np
```

### Complete Workflow Example

```bash
# 1. Prepare the circuit
provekit-cli prepare circuit.json -p scheme.pkp -v scheme.pkv

# 2. Generate a proof
provekit-cli prove scheme.pkp inputs.toml -o proof.np

# 3. Verify the proof
provekit-cli verify scheme.pkv proof.np
```

## Expected Output

During proof generation, you'll see logging information:

```
Read Noir proof scheme: constraints=1234, witnesses=567
```

Successful proof generation produces no error output and returns exit code 0.

If an error occurs:

```
Error: While proving Noir program statement

Caused by:
    Missing required input: x
```

## What Happens During Proving

The prove command:

1. Loads the prover from the PKP file
2. Parses input values from the TOML file
3. Generates witnesses using the input values
4. Computes the R1CS proof using WHIR
5. Serializes and writes the proof to the output file

<Info>
In test builds, the proof is automatically verified after generation to ensure correctness.
</Info>

## Proof File Format

The generated `.np` (Noir Proof) file contains:
- Public inputs
- WHIR R1CS proof
- Commitment data
- Witness polynomial evaluations

The proof file is serialized using Postcard format and can be verified using the [`verify`](/cli/verify) command.

## Performance Considerations

- Proof generation time scales with circuit size (number of constraints)
- Memory usage is proportional to the number of witnesses
- The PKP file is loaded entirely into memory
- Use Tracy profiling (with `--tracy` flag) to identify bottlenecks
