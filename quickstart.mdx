# Quick start

This guide walks you through the complete ProveKit workflow using a real example circuit. You'll learn how to compile a Noir circuit, prepare the proving system, generate proofs, and verify them.

## Overview

The ProveKit workflow consists of four main steps:

1. **Compile** - Compile your Noir circuit to R1CS format
2. **Prepare** - Generate prover and verifier files from the compiled circuit
3. **Prove** - Generate a zero-knowledge proof using witness inputs
4. **Verify** - Verify the generated proof

## Example circuit

We'll use the `poseidon-rounds` example, which demonstrates Poseidon2 hash computation with multiple rounds. This circuit:

- Takes two field elements as input
- Computes Poseidon2 hash iteratively for 1000 rounds
- Verifies the final hash matches the expected result

<CodeGroup>
```noir Circuit (main.nr)
use dep::poseidon2;

fn main(plains: [Field; 2], result: Field) {
    let mut hash = poseidon2::bn254::hash_2(plains);
    let rounds = 1000;
    for _ in 0..rounds {
        hash = poseidon2::bn254::hash_1([hash]);
    }
    println(hash);
    assert(hash == result);
}
```

```toml Input (Prover.toml)
plains = [1, 2]

# Expected result after 1000 rounds
result = '0x0b5b6e4be36ffe606c2f640d972634fcf030bd0346eadcec12d0e3b95bf7f523'
```
</CodeGroup>

## Complete workflow

<Steps>
  <Step title="Navigate to example directory">
    Navigate to the `poseidon-rounds` example directory:

    ```bash
    cd noir-examples/poseidon-rounds
    ```

    This directory contains:
    - `src/main.nr` - The Noir circuit
    - `Prover.toml` - Input values for the prover
    - `Nargo.toml` - Project configuration
  </Step>

  <Step title="Compile the Noir circuit">
    Compile the circuit using `nargo`:

    ```bash
    nargo compile
    ```

    Expected output:
    ```
    [poseidon-rounds] Compiling...
    [poseidon-rounds] Generated ACIR opcodes
    [poseidon-rounds] Saved circuit to ./target/poseidon_rounds.json
    ```

    This generates the compiled circuit in R1CS format at `target/poseidon_rounds.json`.

    <Tip>
      The compiled circuit file contains the R1CS constraint system representation of your Noir program. This file is used by ProveKit to generate the proving and verification keys.
    </Tip>
  </Step>

  <Step title="Prepare the proving system">
    Generate prover and verifier files using ProveKit CLI:

    ```bash
    cargo run --release --bin provekit-cli prepare \
      ./target/poseidon_rounds.json \
      --pkp ./prover.pkp \
      --pkv ./verifier.pkv
    ```

    This command:
    - Reads the compiled circuit from `target/poseidon_rounds.json`
    - Generates the proving key at `prover.pkp`
    - Generates the verification key at `verifier.pkv`

    Expected output:
    ```
    Preparing proving system...
    Circuit loaded: 45,123 constraints
    Generating proving key...
    Generating verification key...
    Prover key written to ./prover.pkp (2.3 MB)
    Verifier key written to ./verifier.pkv (128 KB)
    ```

    <Note>
      The prepare step only needs to be run once per circuit. You can reuse the generated `.pkp` and `.pkv` files for multiple proofs with different inputs.
    </Note>
  </Step>

  <Step title="Generate a proof">
    Generate a zero-knowledge proof using the witness inputs from `Prover.toml`:

    ```bash
    cargo run --release --bin provekit-cli prove \
      ./prover.pkp \
      ./Prover.toml \
      -o ./proof.np
    ```

    This command:
    - Loads the proving key from `prover.pkp`
    - Reads witness values from `Prover.toml`
    - Generates the proof and saves it to `proof.np`

    Expected output:
    ```
    Proving...
    Witness generation: 23ms
    Proof generation: 1,847ms
    Proof written to ./proof.np (18.4 KB)
    ```

    <Tip>
      Use the `--features profiling` flag to enable memory profiling:
      ```bash
      cargo run --release --bin provekit-cli --features profiling prove \
        ./prover.pkp ./Prover.toml -o ./proof.np
      ```
    </Tip>
  </Step>

  <Step title="Verify the proof">
    Verify the generated proof:

    ```bash
    cargo run --release --bin provekit-cli verify \
      ./verifier.pkv \
      ./proof.np
    ```

    This command:
    - Loads the verification key from `verifier.pkv`
    - Reads the proof from `proof.np`
    - Verifies the proof is valid

    Expected output:
    ```
    Verifying proof...
    Proof verification: 47ms
    ✓ Proof is valid
    ```

    If verification fails, you'll see:
    ```
    ✗ Proof verification failed
    ```
  </Step>
</Steps>

## Additional commands

ProveKit CLI provides several utility commands for circuit analysis and debugging.

### Analyze circuit statistics

View detailed statistics about your circuit's R1CS complexity:

```bash
cargo run --release --bin provekit-cli circuit_stats \
  ./target/poseidon_rounds.json
```

Example output:
```
Circuit Statistics:
  Constraints: 45,123
  Public inputs: 1
  Private inputs: 2
  Gates by type:
    - Arithmetic: 42,890
    - Range checks: 2,233
```

### Analyze PKP file size

Breakdown of the prover key file size:

```bash
cargo run --release --bin provekit-cli analyze-pkp ./prover.pkp
```

Example output:
```
PKP File Analysis:
  Total size: 2.3 MB
  R1CS data: 1.8 MB (78%)
  Witness template: 412 KB (18%)
  Metadata: 90 KB (4%)
```

### Show public inputs

Display public inputs with variable names from a proof:

```bash
cargo run --release --bin provekit-cli show-inputs \
  ./verifier.pkv \
  ./proof.np
```

Example output:
```
Public Inputs:
  result: 0x0b5b6e4be36ffe606c2f640d972634fcf030bd0346eadcec12d0e3b95bf7f523
```

### Generate Gnark inputs

Generate inputs for recursive verification in Gnark:

```bash
cargo run --release --bin provekit-cli generate-gnark-inputs \
  ./prover.pkp \
  ./proof.np
```

This creates the necessary files for recursive verification using the Gnark proving system.

## Benchmarking

Compare ProveKit performance against Barretenberg:

```bash
# Prepare the circuit
cargo run --release --bin provekit-cli prepare \
  ./target/poseidon_rounds.json \
  --pkp ./prover.pkp \
  --pkv ./verifier.pkv

# Benchmark with hyperfine
hyperfine \
  'nargo execute && bb prove -b ./target/poseidon_rounds.json -w ./target/poseidon_rounds.gz -o ./target' \
  '../../target/release/provekit-cli prove ./prover.pkp ./Prover.toml'
```

<Note>
  Make sure you have [Barretenberg](https://github.com/AztecProtocol/aztec-packages/blob/master/barretenberg/bbup/README.md) and [hyperfine](https://github.com/sharkdp/hyperfine) installed for benchmarking.
</Note>

## Recursive verification

Verify ProveKit proofs recursively in a Gnark circuit:

```bash
# Navigate to recursive verifier
cd ../../recursive-verifier

# Run Gnark recursive verification
go run cmd/cli/main.go \
  --config ../noir-examples/poseidon-rounds/params_for_recursive_verifier \
  --r1cs ../noir-examples/poseidon-rounds/r1cs.json
```

This enables proof composition and on-chain verification using Gnark's SNARK-friendly proving system.
